# Three-Sponsor Focus

If the goal is to maximize odds of placing, we should probably **not** try to hit everything.

Best strategy:
- choose **1 core primitive sponsor**
- choose **1 differentiator sponsor**
- choose **1 agent-identity / autonomy sponsor**

That gives us a clear story without turning the project into a messy kitchen sink.

## Recommended template

### Sponsor 1 — Core primitive
This is the technical center of the project.
Examples:
- Lido
- Filecoin
- OpenServ
- Uniswap

### Sponsor 2 — Differentiator
This is what makes the build less generic.
Examples:
- MetaMask Delegations
- Venice
- Locus
- Bankr
- ENS

### Sponsor 3 — Agent legitimacy layer
This is the layer that makes it feel native to the hackathon.
Examples:
- ERC-8004 / Agents With Receipts
- Let the Agent Cook
- Self Agent ID

## Best 3-sponsor bundles

### Bundle A — My strongest recommendation
**Lido + MetaMask + ERC-8004**

Why:
- Lido gives the low-level contract primitive
- MetaMask gives the differentiated authority model
- ERC-8004 gives the hackathon-native trust / identity / receipts layer

This is the cleanest, most technically serious combo.

Best framing:
- principal-protected treasury primitive
- delegated sub-budgets / constrained execution
- receipts / agent identity / audit trail

Primary tracks:
- stETH Agent Treasury
- Best Use of Delegations
- Agents With Receipts — ERC-8004

Secondary / umbrella:
- Lido MCP
- Let the Agent Cook
- Open Track

### Bundle B — Best if we want more straightforward execution
**Lido + Bankr + ERC-8004**

Why:
- easier to demonstrate autonomous spend
- easier to show self-funding agent economics
- still keeps a strong contract center

Best framing:
- yield-funded agent operations treasury
- Bankr pays for inference / swaps / execution
- ERC-8004 records identity and work receipts

Primary tracks:
- stETH Agent Treasury
- Best Bankr LLM Gateway Use
- Agents With Receipts — ERC-8004

### Bundle C — Best if we want privacy as the differentiator
**Lido + Venice + ERC-8004**

Why:
- strong sponsor story
- good fit for treasury policy / risk analysis / private decision making
- still contract-real

Best framing:
- private treasury policy engine
- Lido treasury enforces capital boundary
- ERC-8004 proves who acted and what happened

Primary tracks:
- stETH Agent Treasury
- Private Agents, Trusted Actions
- Agents With Receipts — ERC-8004

### Bundle D — Best if we pivot to storage / infra
**Filecoin + ERC-8004 + OpenServ**

Why:
- more infra-native
- strong if storage / logs / state are truly load-bearing
- weaker fit for the current Lido treasury line

Best framing:
- onchain agent state / receipt archive / persistent memory layer
- OpenServ multi-agent coordination
- ERC-8004 identity and provenance

Primary tracks:
- Best Use Case with Agentic Storage
- Ship Something Real with OpenServ
- Agents With Receipts — ERC-8004

### Bundle E — Best if we want the easiest shipping path
**Celo + Bankr + ERC-8004**

Why:
- likely easier than the others
- still aligned with agent economy themes
- may be less distinctive than Lido + MetaMask

Best framing:
- stablecoin-native autonomous service agent
- Bankr funds and runs inference / execution
- ERC-8004 gives trust and verifiability

Primary tracks:
- Best Agent on Celo
- Best Bankr LLM Gateway Use
- Agents With Receipts — ERC-8004

## My recommendation

If we are serious about building something low-level and differentiated, choose:

## **Lido + MetaMask + ERC-8004**

That is the most coherent three-sponsor story.

If we want the easier fallback, choose:

## **Lido + Bankr + ERC-8004**

## Important anti-pattern

Do **not** try to make all three sponsors equally central.

Instead:
- one sponsor owns the primitive
- one sponsor owns the differentiator
- one sponsor owns trust / legitimacy

That yields a project that sounds designed rather than assembled.

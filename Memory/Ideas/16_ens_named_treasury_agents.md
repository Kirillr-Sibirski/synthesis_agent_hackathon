# Idea 7 — ENS-Named Treasury Agents

## One-liner
Treasury accounts, operators, and sub-agents are all surfaced through ENS identities so humans never manage raw addresses while configuring agent budgets and permissions.

## Why it looks strong
This is more UX-forward, but still useful as a layer on top of a treasury primitive.

## Core primitive
- Treasury vault gets ENS identity
- Sub-agents get ENS names
- Delegation targets and budget recipients can be configured by ENS
- Receipts and reputation events resolve to readable names

## Best target tracks
- ENS Identity
- ENS Open Integration
- ENS Communication
- stETH Agent Treasury
- Agents With Receipts — ERC-8004
- Open Track

## Novelty
Not huge by itself, but helpful as an additive angle if the core project already has treasury + delegation logic.

## Risk
Not strong enough as the sole main idea.

## Build scope
Low.
